# v0.3 双层 HPSO 版本说明

`v0.3` 是一次算法主线重构版本。正式主模型从 `v0.25` 的“周度 PPO + 小时级显式规则修正”调整为“双层混合粒子群算法 HPSO + 15 分钟代理结算回测”。上层用 HPSO 决定周度中长期合约调整量并保留现货边际敞口带宽诊断字段，下层用 HPSO 在周内逐小时优化现货合约修正量，最终仍通过 15 分钟代理结算层评价成本、风险和套保效果。

## 版本边界

- 正式主模型改为双层 HPSO，不再以 PPO 作为主训练算法。
- 下层不再以显式规则作为主决策机制，而是由 HPSO 搜索小时级 `delta_q`。
- HPSO 口径固定为“PSO + 退火扰动 + BP 局部精修”，不采用遗传算子混合。
- 15 分钟实时市场仍只作为结算与回测层，不扩展为第三层独立优化。
- `dynamic_lock_only` 继续作为强基准参与比较，并作为上层合约调整量的诊断基准。
- 中长期合约调整量不再按 `delta_lock_cap`、`delta_h_max`、`lock_ratio_min` 或 `lock_ratio_max` 硬裁剪。
- 现货合约修正量不再按非负、带宽、平滑或辅助服务耦合规则硬裁剪；这些字段仅保留为诊断。
- 从本版本开始，每个 `v*.md` 必须写入当版实际使用的数学公式。

## 双层算法结构

### 上层：周度中长期持仓 HPSO

- `delta_lock_ratio_raw`：相对 `dynamic_lock_only` 基准的中长期合约调整量，不再归一化裁剪；
- `lock_ratio_final`：`lock_ratio_base + delta_lock_ratio_raw`，不再做硬边界裁剪；
- `exposure_bandwidth`：保留为下层诊断字段，不再作为现货合约硬边界。

上层继续保留 `lock_ratio_base`、`delta_lock_ratio` 和 `lock_ratio_final`，用于说明 HPSO 结果相对强基准的偏离程度。制度状态仍按生效时间进入特征与结算口径。

### 下层：小时级现货修正 HPSO

下层在给定周度 `lock_ratio_final` 与 `exposure_bandwidth` 后，对周内小时级现货合约修正量进行滚动优化。

下层粒子变量为小时级 `delta_q` 序列，含义是在基础现货需求上进行加减修正。输出需能形成完整小时轨迹，并映射到 15 分钟代理结算。

下层不再对 `delta_q` 或 `q_spot` 执行非负、边际敞口带宽、小时级平滑、辅助服务耦合硬裁剪。价格偏差、负荷偏差、新能源偏差仍进入目标函数风险项，带宽和平滑字段保留在轨迹表中用于诊断。

## HPSO 机制

HPSO 以标准粒子群更新为主体，并在迭代过程中加入退火扰动和 BP 局部精修：

- 粒子速度由惯性项、个体最优项和群体最优项共同更新；
- 退火温度随迭代递减；
- 扰动幅度随温度下降而收缩；
- 停滞时允许通过扰动跳出局部最优；
- 对可微目标函数执行若干步反向传播局部精修，精修后仅接受目标值改善的粒子位置；
- 固定随机种子时必须保证搜索过程可复现。

建议配置项包括粒子数、迭代数、惯性权重、认知学习因子、社会学习因子、退火初温、降温率、扰动强度、停滞判定窗口、BP 步数、BP 学习率、BP 梯度裁剪阈值和随机种子。

## 本版数学公式

后续每个版本说明必须保留同等粒度的公式记录；如果代码公式变化，必须同步更新对应版本文件。

### 1. HPSO 更新式

对粒子 \(i\) 在第 \(t\) 轮的位置 \(x_i^t\)、速度 \(v_i^t\)、个体最优 \(p_i^t\)、全局最优 \(g^t\)，标准 PSO 更新为：

\[
v_i^{t+1}=\omega v_i^t+c_1 r_1\odot(p_i^t-x_i^t)+c_2 r_2\odot(g^t-x_i^t)
\]

\[
\tilde{x}_i^{t+1}=x_i^t+v_i^{t+1}
\]

退火扰动为：

\[
T_t=T_0\rho^t,\qquad
x_i^{t+1,0}=\tilde{x}_i^{t+1}+\epsilon_i\odot s\cdot \lambda T_t\cdot \kappa_t
\]

其中 \(s\) 是初始化搜索尺度，\(\lambda\) 是扰动强度，\(\kappa_t\) 在停滞时取较大值。对设置为无硬限制的合约维度，\(x\) 不再做上下界裁剪；有限边界仅用于初始化尺度。

BP 精修对每个粒子执行：

\[
x_i^{t,k+1}=x_i^{t,k}-\eta\cdot \operatorname{clip}(\nabla_x J(x_i^{t,k}), G)
\]

若 \(J(x_i^{t,K}) < J(x_i^{t,0})\)，接受精修后位置；否则保留精修前位置。

### 2. 上层合约目标

上层变量为：

\[
a_w=(\delta h_w,b_w)
\]

其中 \(\delta h_w\) 是不设硬限的中长期合约调整量，\(b_w\) 是诊断性敞口带宽。最终锁定比例为：

\[
h_w=h_w^{base}+\delta h_w
\]

中长期合约量与现货基准量为：

\[
Q_w^{LT}=h_w\hat{L}_w,\qquad Q_w^{spot}=\hat{L}_w-Q_w^{LT}
\]

上层目标函数为：

\[
J_U=\sum_w(Q_w^{LT}P_w^{LT}+Q_w^{spot}P_w^{DA})
+\alpha_r\sum_w |Q_w^{spot}|(|S_w|+100R_w)
+\alpha_{\Delta h}\sum_w |h_w-h_{w-1}|\bar{L}
+\alpha_b\sum_w |h_w-h_w^{base}|\bar{L}
+\alpha_c\sum_w\left(b_w\hat{L}_w+|\delta h_w|\bar{L}+\gamma_h\delta h_w^2\bar{L}\right)
\]

其中 \(S_w\) 是价差风险代理，\(R_w\) 是新能源扰动代理，\(\bar{L}\) 是周负荷均值尺度，\(\gamma_h\) 对应 `hpso.objective_weights.contract_adjustment_quadratic`。最后一项是软交易/调整成本，用于替代硬边界并防止无界解。

### 3. 下层现货合约目标

下层变量为逐小时现货修正量：

\[
\Delta q_t
\]

基础现货需求与执行现货量为：

\[
q_t^{base}=\hat{l}_t-q_t^{LT},\qquad q_t^{spot}=q_t^{base}+\Delta q_t
\]

目标跟踪量为：

\[
q_t^{target}=l_t^{ID}-q_t^{LT}
\]

下层目标函数为：

\[
J_L=\sum_t q_t^{spot}P_t^{DA}
+\beta_r\sum_t |\Delta q_t|\left(|S_t|+0.1|D_t^L|+0.05|D_t^R|\right)
+\beta_s\sum_t|\Delta q_t-\Delta q_{t-1}|
+\beta_e\sum_t|q_t^{spot}-q_t^{target}|
+\beta_c\sum_t\left(|\Delta q_t|+\gamma_q\Delta q_t^2\right)
\]

其中 \(\gamma_q\) 对应 `hpso.objective_weights.spot_delta_quadratic`。下层不再对 \(q_t^{spot}\) 执行非负裁剪，不再对 \(\Delta q_t\) 执行带宽裁剪或平滑裁剪；这些量仅进入诊断字段。

### 4. 15 分钟代理结算

小时合约量映射到 15 分钟后：

\[
E_{\tau}^{LT}=0.25q_t^{LT},\qquad E_{\tau}^{spot}=0.25q_t^{spot}
\]

\[
E_{\tau}^{sch}=E_{\tau}^{LT}+E_{\tau}^{spot}
\]

\[
E_{\tau}^{imb}=|E_{\tau}^{actual}-E_{\tau}^{sch}|
\]

15 分钟采购成本为：

\[
C_\tau=E_{\tau}^{LT}P_w^{LT}+E_{\tau}^{spot}P_\tau^{DA}+E_{\tau}^{imb}P_\tau^{ID}\mu
\]

其中 \(\mu\) 是偏差惩罚倍率。总成本、波动率、CVaR 和套保误差均基于该 15 分钟结算轨迹汇总。

## CUDA 与本地环境

v0.3 的 HPSO 搜索应优先使用 CUDA 12.8 加速。粒子位置、速度、个体最优、群体最优、退火扰动和目标函数批量评估应尽量使用 `torch.Tensor` 在 GPU 上完成，只有文件读写、报告生成和少量表格整理保留在 CPU/Pandas 路径。

本地正式运行环境固定为 `torch311`：

```bash
mamba activate torch311
pip install -r requirements.txt
```

运行入口同步使用 `mamba run -n torch311 python -m src.scripts.run_pipeline`。若 CUDA 不可用，正式 HPSO 实现应显式报错或降级到受控 CPU 路径，并在运行摘要中写明实际设备。

## 目标函数

v0.3 主目标改为直接最小化综合成本风险，不再使用原 PPO 三层 reward 作为主训练目标。

综合目标至少包括采购成本、风险项、交易成本、周度持仓变动软惩罚、小时级平滑软惩罚和套保误差惩罚。硬约束违约惩罚不再用于合约与现货合约模型输出。

原 PPO reward 可作为历史版本说明保留，不再作为 v0.3 主模型的优化函数。

## 输出与对比

正式输出继续写入：

`outputs/v0.3/<真实输出>`

至少应保留 HPSO 最优周度动作表、小时级 `delta_q` 轨迹、15 分钟代理结算明细、粒子搜索收敛曲线、搜索参数快照、特征清单、滚动验证摘要、政策文件清单、结构化规则表、解析失败清单，以及与 `dynamic_lock_only`、固定持仓、无现货修正、简单滚动对冲等基准策略的比较表。

结果报告至少比较累计采购成本、均值、波动率、最大回撤、CVaR、套保误差、平均调节电量和相对基准差异。

## 不变项

- 研究主体仍为湖南电力市场售电公司。
- 研究期仍为 2025-11-01 至 2026-03-20。
- 主结构仍为“周度中长期决策 + 小时级现货修正 + 15 分钟代理结算回测”。
- 政策规则必须区分发布时间与生效时间。
- 2026-01-01 新能源机制电价执行口径不得前移。
- 2026-02 起中长期价格机制变化不得用于 2025-11 至 2026-01。
- 不引入需求响应、虚拟投标、调频收益、金融输电权或发电侧独立优化。

## 验证要求

- 上层 HPSO 输出不得被锁定比例边界、周度变动约束或 `delta_lock_cap` 硬裁剪改写。
- 下层 HPSO 输出不得被非负、带宽、平滑或辅助服务耦合硬裁剪改写。
- 轨迹表仍需输出带宽、平滑、裁剪计数字段，裁剪计数在主 HPSO 路径应为 0。
- 固定随机种子下，退火扰动与最终结果必须可复现。
- 15 分钟结算字段应与既有回测口径保持可比。
- 回测必须保留 `dynamic_lock_only` 强基准，不能只报告 HPSO。
- 制度切换点必须按生效时间进入模型，重点检查 2026-01-01 与 2026-02-01。
