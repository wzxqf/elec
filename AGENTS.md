# AGENTS.md

## 项目定位
本项目服务于论文《电力资产主动管理框架——枯水期DRL驱动下中长期合约择时与现货风险对冲策略研究》的程序实现部分。研究主体为湖南电力市场中的售电公司，研究期为 2025-11-01 至 2026-03-20，核心任务是构建“周度中长期持仓决策 + 小时级现货滚动修正”的多时间尺度动态决策框架。:contentReference[oaicite:0]{index=0}

## 总体目标
程序应围绕以下目标展开：
1. 以上层周度决策确定中长期目标净持仓与锁定比例；
2. 以下层小时级决策根据负荷偏差、现货价格和新能源波动进行滚动修正；
3. 通过回测比较策略在采购成本、收益稳定性和风险控制方面的表现。:contentReference[oaicite:1]{index=1}

## 模型边界
- 研究主体固定为售电公司，不扩展到发电侧、新能源场站或电网调度侧独立优化。
- 核心市场范围仅包括：中长期市场、日前现货市场、日内/实时现货市场。
- 15分钟实时市场只作为结算与回测层，不单独训练一套独立决策模型。
- 调频辅助服务、需求响应、虚拟投标、金融输电权、调频容量市场不进入主模型动作空间。:contentReference[oaicite:2]{index=2}

## 制度口径
程序实现必须严格服从研究期内的制度时序，不得混用政策。

### 持续适用的基础规则
- 《湖南省电力中长期交易实施细则》
- 《湖南省电力现货市场交易实施细则》
- 《关于进一步加强湖南电力辅助服务市场建设有关事项的通知》
- 《湖南省深化新能源上网电价市场化改革促进新能源高质量发展实施方案》
- 《湖南省新能源机制电价竞价实施细则》

### 分时进入分析的规则
- 《关于2025年度新能源机制电价竞价工作有关事项的通知》中，“2025年投产且纳入机制的项目，机制电价自2026年1月1日起执行”，该口径只能自 2026-01-01 起进入模型。
- 《关于完善2026年度电力中长期交易价格机制的通知》中，年度中长期合同“40%固定价格 + 60%灵活价格”的安排自 **2026年2月份年度多月交易开始执行**，不得提前用于 2025-11 至 2026-01 的环境设定。

### 必守原则
- 所有制度分析必须区分“发布时间”与“生效时间”。
- 后期政策不得前移覆盖前期制度环境。
- 如果规则与时间不匹配，以生效时点为准，不以文件日期机械替代。:contentReference[oaicite:6]{index=6}

## 市场机制约束
### 中长期与现货衔接
湖南现货规则明确采用“中长期合约仅作为结算依据管理市场风险、现货交易全电量集中竞价”的模式，中长期交易结果不作为调度执行依据。程序建模时不得把中长期合约直接当作物理执行计划。

### 中长期交易时间粒度
中长期交易按每天24小时划分为24个时段，涵盖年度、月度和月内（多日）交易。程序中的中长期合约曲线、分解和结算口径应按此处理。

### 日前现货时间粒度
日前现货市场按日组织，运行日共有96个15分钟时段。若程序使用小时级决策，可在内部聚合为小时，但结算与回测必须能还原到15分钟粒度或兼容96点结果。

### 零售侧约束
零售用户在同一合同周期内只能向一家售电公司购电，同一结算周期仅可生效一个零售套餐。零售电价机制应作为售电公司收益端约束，而非自由可调参数。

## 推荐建模口径
### 上层：周度中长期持仓决策
- 决策变量：下一周净持仓水平、锁定比例、必要的分时持仓结构。
- 状态变量应至少包含：历史现货价格、中长期成交价、负荷预测偏差、市场波动指标、政策阶段标记、新能源相关扰动代理变量。
- 目标函数应体现收益与风险的联合权衡，而不是只追求平均收益。:contentReference[oaicite:11]{index=11}

### 下层：小时级现货滚动修正
- 在既定持仓基础上，对日前/日内市场进行滚动修正。
- 重点处理：现货—合约价差风险、负荷偏差风险、新能源波动带来的敞口变化。
- 下层不单独训练另一套复杂DRL作为主线，默认采用轻量化滚动修正机制或规则/优化混合方式。:contentReference[oaicite:12]{index=12}

## 明确禁止
- 不得改回“月度中长期决策”旧方案，当前版本以周度为准。:contentReference[oaicite:13]{index=13}
- 不得引入 LLM 政策评分系统、在线学习、SAC、PSR、独立下层DRL训练、2015—2023长期历史典型场景回测。:contentReference[oaicite:14]{index=14}
- 不得把需求响应、调峰辅助服务、调频市场收益直接并入主策略收益函数。
- 不得自行补充论文未设定的新市场机制、新主体或新数据字段。

## 数据与实现要求
- 所有数据处理都要保留“制度时点标签”。
- 训练集、验证集、回测集划分必须避免未来信息泄漏。
- 缺失值、异常值、极端值处理必须可追踪、可复现。
- 输出结果至少包括：累计收益、均值、波动率、最大回撤、尾部风险指标、与基准策略比较结果。:contentReference[oaicite:15]{index=15}

## 基准策略
程序中必须保留可比较的基准策略，例如：
- 固定持仓策略
- 经验规则策略
- 不进行现货修正的静态策略
- 简单滚动对冲策略

不得只报告DRL策略结果。:contentReference[oaicite:16]{index=16}

## 输出要求
- 优先保证模型设定与论文规划一致，其次再考虑算法复杂度。
- 所有图表、日志、结果文件命名清晰，避免临时脚本堆叠。
- 每次修改核心逻辑时，同步更新参数说明与实验配置说明。
- 最终代码应支持复现实验，而不是只输出一次性结果。

## 开发原则
- 先保证制度一致性，再保证数值效果。
- 先做可运行的最小闭环，再逐步补充细节。
- 保持模块化：数据层、环境层、策略层、回测层、评估层分开。
- 尽量少做与论文主线无关的工程扩展。
