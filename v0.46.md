# elec 项目 v0.46 发布说明

## 1. 版本定位

`v0.46` 是当前唯一正式口径，对应：

- `project.version: v0.46`
- `training.algorithm: HYBRID_PSO_V040`
- `outputs/v0.46/`

本轮重点不是扩展新机制，而是把项目文档、配置、运行链、测试和版本化产物统一到当前正式实现。

## 2. 当前正式参数

当前正式参数以根配置为准：

- 上层 `contract_curve_latent.size = 20`
- 上层 `action_head.size = 8`
- 下层四组响应 `response_size = 12`
- HPSO 优化器 `init_scale/inertia/cognitive/social/position_clip_abs = 0.12 / 0.68 / 1.30 / 1.45 / 1.25`
- 上层搜索预算 `192 x 320`，真实维度 `186`
- 下层搜索预算 `160 x 320`，真实维度 `48`

## 2.1 现货修正口径补丁

- `spot_hedge_mwh` 是带符号的小时级净修正量，进入结算时保留方向。
- 交易摩擦、平均修正量和成交活跃度仍按 `abs(spot_hedge_mwh)` 统计。
- 小时级修正通过 `hourly_gate` 过滤低边际收益信号。
- 训练奖励对比 `[0.50, 0.55, 0.60]` 合约比例强基准族。

## 5. 验证口径

正式验证改为 Jupyter 远程执行：本地只负责打包上传当前工作区，服务器 `torch311` kernel 内运行全流程和测试，完成后再拉回服务器产物。

```powershell
$env:ELEC_JUPYTER_URL="http://10.26.27.72:9007/"
$env:ELEC_JUPYTER_PASSWORD="<由老师提供的 Jupyter 密码>"
$env:ELEC_JUPYTER_KERNEL="python3"
$env:ELEC_REMOTE_ENV="torch311"
D:\miniforge\envs\torch311\python.exe run_remote_jupyter.py --probe
D:\miniforge\envs\torch311\python.exe run_remote_jupyter.py
```

默认远程命令顺序为：先解析 `ELEC_REMOTE_ENV=torch311` 的 Python，再执行 `python run_all.py`，随后执行 `python -m src.scripts.run_pytest tests -q`。当前服务器返回的 Jupyter kernel spec 为 `python3`；`--probe` 会打印 kernel Python 路径和项目运行 Python 路径，用于确认远程 `torch311` 环境是否可解析。当前已纳入自动发现的实测路径为 `/research/miniforge3/envs/torch311/bin/python`；若路径变化，可用 `--remote-python /path/to/torch311/bin/python` 覆盖自动发现。拉回结果位于 `outputs/v0.46/remote_jupyter/<run_id>/`；完整远程 pipeline 返回码为 0 时，会同步覆盖本地正式 `outputs/v0.46` 结果区，`remote_jupyter/` 运行记录保留。`--skip-pipeline` 不会覆盖本地正式结果；调试时可加 `--no-sync-local-output` 禁止覆盖。
