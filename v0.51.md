# elec 项目 v0.51 发布说明

## 1. 版本定位

`v0.51` 是当前唯一正式口径，对应：

- `project.version: v0.51`
- `training.algorithm: HYBRID_PSO_V040`
- `outputs/v0.51/`

本轮重点是承接 `v0.47` 审计修复结果，并把项目文档、配置、运行链、测试和版本化产物统一到当前正式实现。

## 2. 当前正式参数

当前正式参数以根配置为准：

- 上层 `contract_curve_latent.size = 20`
- 上层 `action_head.size = 8`

- 上层搜索预算 `192 x 320`，真实维度 `182`
- 下层搜索预算 `160 x 320`，真实维度 `48`

## 2.1 现货修正口径补丁

- `spot_hedge_mwh` 是带符号的小时级净修正量，进入结算时保留方向。
- 交易摩擦、平均修正量和成交活跃度仍按 `abs(spot_hedge_mwh)` 统计。
- 小时级修正通过 `hourly_gate` 过滤低边际收益信号。
- 训练奖励对比 `[0.50, 0.55, 0.60]` 合约比例强基准族。

## 2.2 审计修复后口径

- 2026-02 起中长期价格联动生成 `lt_price_w_effective`，并同步进入评分核、物化回测和基准策略使用的 `lt_weekly_price` 张量。
- `release_manifest.json`、`run_manifest.json` 和 `artifact_index.md` 覆盖模型、训练、验证、滚动回测、基准、消融、稳健性、人工报告和 AI 结构化报告等全流程产物。
- 每次完整 pipeline 运行后只在 `reports/` 下生成 `v0.51_human_report.md` 与 `v0.51_ai_structured_report.json` 两份入口报告；CSV、模型、日志、配置快照和布局文件统一归入 `raw/`。
- 滚动外推窗口、正式验证集和正式测试集分开报告；基准比较使用正式验证集与正式测试集合并后的持出口径。
- 稳健性由后处理扰动改为对滚动测试窗口重跑情景结算，输出 `scenario_settlement_method=rerun_materialize_particle_pair`。
- `settle_week()` 仅作为参考实现保留，当前主链统一为 `score_kernel.py -> materialize.py`。

其中面向人的报告统一命名为 `reports/v0.51_human_report.md`。

## 5. 验证口径

正式验证改为 Jupyter 远程执行：本地只负责打包上传当前工作区，服务器 `torch311` kernel 内运行全流程和测试，完成后再拉回服务器产物。

```powershell
$env:ELEC_JUPYTER_PASSWORD="<本机环境变量提供的 Jupyter 密码>"
.\run_remote_jupyter.ps1
```

PowerShell 入口会自动解析本机 `torch311` Python，补齐 Jupyter URL、kernel 和远程环境的非敏感默认值，默认先执行 `--probe`，确认远程 `torch311` Python 后再上传运行。默认远程命令顺序为：先解析 `ELEC_REMOTE_ENV=torch311` 的 Python，再执行 `python run_all.py`，随后执行 `python -m src.scripts.run_pytest tests -q`。当前服务器返回的 Jupyter kernel spec 为 `python3`；当前已纳入自动发现的实测路径为 `/research/miniforge3/envs/torch311/bin/python`；若路径变化，可用 `-RemotePython /path/to/torch311/bin/python` 覆盖自动发现。拉回结果位于 `outputs/v0.51/raw/remote_jupyter/<run_id>/`；完整远程 pipeline 返回码为 0 时，会同步覆盖本地正式 `outputs/v0.51` 结果区，`raw/remote_jupyter/` 运行记录保留。`-SkipProbe` 可跳过预检，`-DryRun` 只打印运行计划。
