# v0.45 Architecture And Implementation

## 1. 版本定位

`v0.45` 是当前唯一正式口径。项目围绕湖南售电公司在 `2025-11-01` 至 `2026-03-20` 研究期内的多时间尺度采购与风险控制展开，当前正式主线固定为：

- 周度中长期头寸参数化搜索
- 小时级现货边际修正
- 15 分钟代理结算回测

唯一人工入口是根目录 `experiment_config.yaml`，唯一正式输出目录是 `outputs/v0.45/`。

## 2. 总体执行链

当前真实运行链如下：

`run_all.py -> src.scripts.run_pipeline -> prepare_project_context -> train -> evaluate -> backtest -> reports/manifests`

各阶段职责如下：

- `run_all.py`：解析 `project.version`、定位 `torch311` Python、创建版本化日志目录并启动全流程。
- `src.scripts.run_pipeline`：串联训练、验证、回测与总报告生成。
- `src.scripts.common.prepare_project_context`：构造运行上下文、审计产物、张量包、滚动计划与 manifest。
- `src.scripts.train`：执行双层 HPSO 训练并落盘模型与训练摘要。
- `src.scripts.evaluate`：在验证集物化执行轨迹、收益风险指标和模块化分析结果。
- `src.scripts.backtest`：执行滚动再训练回测、基准比较、消融、稳健性与约束激活审计。

## 3. 模块结构

### 3.1 数据层

- `src/data/loader.py`：定位并读取真实 `total.csv`。
- `src/data/preprocess.py`：做频率校验、缺失处理、质量报告。
- `src/data/weekly_builder.py`：把 15 分钟原始数据编译为周度、小时级和 15 分钟研究对象。
- `src/data/feature_engineering.py`：生成周度/小时级衍生特征。

数据层输出的核心对象包括：

- `weekly_metadata`
- `weekly_features`
- `hourly`
- `quarter`
- `feature_manifest`

### 3.2 政策与制度层

- `src/policy/policy_parser.py`：扫描 `政策环境/`，生成文件清单、结构化规则表和失败清单。
- `src/policy_deep/document_reader.py`：把规则说明整理为 `document_units`。
- `src/policy_deep/regime_builder.py`：生成当前制度状态和四组前瞻制度状态。
- `src/policy/market_constraints.py`：把研究期市场制度约束整理为模型对齐报告。
- `src/policy/feasible_domain.py`：把制度状态编译为周度/小时级动作边界和结算语义。
- `src/policy/projection.py`：对周度动作和小时级现货修正执行正式投影。

制度链路的正式输出包括：

- `policy_file_inventory.csv`
- `policy_document_units.csv`
- `policy_rule_table.csv`
- `policy_parse_failures.csv`
- `policy_state_trace.csv`
- `feasible_domain_manifest.csv`
- `market_rule_constraints.csv`

### 3.3 参数布局与张量层

- `src/model_layout/compiler.py`：把 `parameter_compiler` 编译成上下层参数块布局。
- `src/model_layout/schema.py`：定义参数块和切片结构。
- `src/training/tensor_bundle.py`：把周度特征、政策状态、小时特征、15 分钟价格、可行域边界编译成 Torch 张量包。

当前正式参数布局分为：

- 上层：`weekly_feature_weights`、`policy_feature_weights`、`contract_curve_latent`、`action_head`
- 下层：`spread_response`、`load_deviation_response`、`renewable_response`、`policy_shrink_response`

## 4. 训练与评分机理

### 4.1 HPSO 主体

- `src/agents/hybrid_pso.py`：维护上下层粒子、速度、个体最优和全局最优，输出模型、运行时摘要和训练轨迹。

当前训练器的职责是：

- 读取上下层真实维度
- 在统一设备上初始化粒子
- 调用评分核批量计算目标值
- 更新双层粒子并记录最优结果

### 4.2 评分核

- `src/training/score_kernel.py`：把上层粒子、下层粒子和 `TrainingTensorBundle` 联结起来，直接产出经济量、风险量和约束量。

评分核中的关键计算路径是：

1. 用周度特征与政策状态计算 `contract_adjustment_mwh_raw` 和 `exposure_band_mwh_raw`
2. 用 `feasible_domain` 做周度投影，得到 `contract_adjustment_mwh_exec`、`exposure_band_mwh`、`contract_position_mwh`
3. 用 `contract_curve_latent` 生成 24 小时合约曲线
4. 用价差、负荷偏差、新能源扰动和时段权重生成小时级 `raw_spot_hedge_mwh`
5. 用 `project_hourly_hedge_tensor()` 做小时级 share/ramp 投影
6. 在 15 分钟结算层计算采购成本、收益、`CVaR99`、套保误差和约束惩罚
7. 以 `excess_profit - tail - hedge - friction - violation` 形成训练奖励

小时级 `spot_hedge_mwh` 保留正负方向。正值表示增加现货侧计划电量，负值表示减少现货侧计划电量；结算计划电量按 `max(contract_position + net_spot_hedge, 0)` 计算。摩擦成本和成交活跃度按绝对成交量统计。

### 4.3 基准口径

正式比较对象保留在分析与回测层，不进入单独训练主线。当前默认强基准是 `dynamic_lock_only`，同时输出固定持仓、静态无现货修正和简单滚动对冲等结果。

## 5. 物化、验证与回测

### 5.1 物化层

- `src/backtest/materialize.py`：把最优上下层粒子还原为周度结果、小时结果和 15 分钟代理结算结果。

核心输出字段包括：

- 周度：`contract_adjustment_mwh_raw`、`contract_adjustment_mwh_exec`、`contract_position_mwh`、`exposure_band_mwh`、`profit_w`、`reward_w`、`cvar99_w`
- 小时级：`spot_hedge_mwh`、`spot_hedge_limit_mwh`
- 结算层：`scheduled_energy_15m`、`actual_need_15m`、`procurement_cost_15m`

### 5.2 验证层

- `src/scripts/evaluate.py`：在验证集物化主策略，并补充：
  - `contract_value_weekly.csv`
  - `risk_factor_manifest.csv`
  - `policy_risk_adjusted_metrics.csv`
  - `validation_summary.md`

### 5.3 回测层

- `src/scripts/backtest.py`：按滚动计划重训窗口模型，并输出：
  - `rolling_window_schedule.csv`
  - `rolling_parameter_snapshots.csv`
  - `rolling_validation_metrics.csv`
  - `rolling_backtest_metrics.csv`
  - `rolling_weekly_results.csv`
  - `rolling_hourly_results.csv`
  - `rolling_settlement_results.csv`
  - `benchmark_comparison.csv`
  - `ablation_metrics.csv`
  - `robustness_metrics.csv`

## 7. 当前实现边界

`v0.45` 当前实现明确保持以下边界：

- 不扩展新的市场主体
- 不增加独立训练主线
- 不把中长期头寸当作物理执行计划
- 不绕过 `feasible_domain -> projection` 直接下发业务动作
- 不允许非版本化输出目录重新进入正式链路
