# CONSTRAINTS.md

## v0.4 生效约束

本文件只保留当前版本有效约束。任何 PPO、SB3、gym、旧静态参数推断链、`lock_ratio` 主语义和动作裁剪链历史口径均已失效。

## 一、研究对象边界

- 研究主体固定为湖南电力市场售电公司
- 研究期固定为 2025-11-01 至 2026-03-20
- 主线固定为周度参数化中长期持仓策略、小时级现货滚动修正、15 分钟代理结算回测
- 不扩展到发电侧、新能源场站、电网调度侧独立优化

## 二、算法边界

- 主算法固定为参数化双层混合粒子群 `HYBRID_PSO_V040`
- 上层粒子直接表示周度参数化持仓策略
- 下层粒子直接表示小时级滚动修正规则
- 训练热路径固定为 Torch 张量评分，不得回退到 pandas 串行主链
- 不再使用旧强化学习训练栈或旧静态参数推断主链
- 不引入独立下层 DRL

## 三、约束边界

- 合约调整量只经政策规则投影约束
- 边际敞口与小时级现货修正必须先进入 `feasible_domain -> projection`
- 不得再对业务决策使用额外极端裁剪
- 不得使用 `lock_ratio_min`、`lock_ratio_max`、`delta_lock_cap`
- 不得使用 `bound_clip_count`、`smooth_clip_count`、`soft_clip_count`、`non_negative_clip_count` 作为正式业务字段
- 允许保留的仅是数值稳定性保护，不得将其伪装成业务约束

## 四、训练与回测边界

- 训练、验证、回测、滚动再训练调度必须显式分离
- 主回测必须使用滚动再训练或扩展窗再训练
- 不接受“训练一次后整段测试集静态推断”作为主回测方式
- 训练阶段不得每轮生成大量 DataFrame 并 concat
- 报告阶段才允许物化可解释表格与审计文件

## 五、制度边界

- 必须区分政策发布时间与生效时间
- 后期政策不得前移覆盖前期制度环境
- 政策目录必须产出文件清单、元数据索引、结构化规则表、解析失败清单
- 制度状态不得退化为简单哑变量，必须保留当前制度状态和四组前瞻状态
- LLM 候选规则解析必须可关闭、可缓存，不得让训练主链依赖在线服务

## 六、市场机制边界

- 中长期合约仅作为结算依据管理市场风险，不得当作物理执行计划
- 中长期交易口径按 24 小时曲线处理
- 日前现货可在训练期内部聚合到小时，但回测结算必须兼容 96 点结果
- 零售电价机制作为收益端约束，不是自由动作变量

## 七、收益与风险边界

- 经济收益函数 `profit` 与训练奖励函数 `reward` 必须分离定义
- 所有正式 `CVaR` 统计默认指 `CVaR_99%`
- 不得以 `reward = -procurement_cost` 代替正式奖励函数
- 必须以 `dynamic_lock_only` 作为正式强基准

## 八、输出边界

正式产物必须写入 `outputs/<version>/...`，其中 `<version>` 取自 `project.version`。至少输出：

- 参数快照
- 特征清单
- 滚动窗口计划
- 每窗口训练/验证/测试指标
- 总体回测汇总
- 政策来源文件数与解析失败文件数
- `profit_w`、`reward_w`、`cvar99_w` 以及参数化电量字段

## 九、唯一正式入口

只保留以下入口：

- `python -m src.scripts.train`
- `python -m src.scripts.evaluate`
- `python -m src.scripts.backtest`
- `python -m src.scripts.run_pipeline`
- `python run_all.py`
