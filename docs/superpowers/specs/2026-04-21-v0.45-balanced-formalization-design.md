# v0.45 Balanced 正式化与归档设计

**Date:** 2026-04-21

**Scope:** 将 `v0.45-param-opt-balanced` 已验证优于旧正式参数的配置收口为当前正式实验入口；以注释形式在根配置中保留 `explore` 备选参数；同步归档旧版本、试验专用配置、试验产物和不再服务正式入口的测试与缓存，确保项目根目录只保留“当前正式版本 + 当前正式测试 + 当前正式结果口径”。

---

## 1. 背景与目标

当前项目同时存在三类并行状态：

1. 根配置 `experiment_config.yaml` 仍代表旧的 `v0.45` 正式参数。
2. `configs/experiments/` 中保留了 `v0.45_param_opt_balanced.yaml` 与 `v0.45_param_opt_explore.yaml` 两套参数试验入口。
3. `outputs/` 根目录同时保留旧正式结果、参数试验结果和当前正式结果，`tests/` 主目录也仍混有服务于试验入口的测试约束。

前面的比对结论已经明确：

- `v0.45-param-opt-balanced` 相比 `v0.44` 旧参数在滚动回测上明显更优；
- `v0.45-param-opt-explore` 不优于 `balanced`，不应升级为正式入口；
- 当前正式主线应当从“旧正式参数”切换到“balanced 参数”，但不应通过直接改名实验产物来伪造正式结果。

本次设计的目标是完成以下四项收口：

1. 根配置切换为 `balanced` 参数，成为唯一正式人工入口。
2. `explore` 参数只作为注释保留在根配置中，不再作为正式运行入口。
3. 旧版本与试验专用配置、试验结果、无效缓存与无效测试归档到 `已归档/`。
4. README、版本说明和 changelog 同步更新，确保版本叙事与目录状态一致。

---

## 2. 明确决策

### 2.1 正式版本标识不变

根配置中的正式版本号继续使用：

- `project.version: v0.45`

不将正式版本号改为 `v0.45-param-opt-balanced`。  
原因是 `balanced` 已被接受为当前正式参数，而不是新的试验分支版本。

### 2.2 正式参数来源切换

根配置 `experiment_config.yaml` 中与参数容量、PSO 搜索规模直接相关的参数，升级为当前 `balanced` 口径。

换言之：

- 正式入口的行为以原 `configs/experiments/v0.45_param_opt_balanced.yaml` 为准；
- 旧 `v0.45` 默认参数不再保留为主线运行参数。

### 2.3 `explore` 仅作注释保留

`explore` 不再保留独立正式配置入口。  
其参数值仅在根配置对应段落以下列方式保留：

- 使用连续注释块；
- 明确标注“探索性参数，未升级为正式入口，仅用于历史比较与后续人工参考”；
- 不参与默认加载，不被任何正式测试视为必需配置文件。

### 2.4 不直接提升实验结果目录为正式结果目录

不将 `outputs/v0.45-param-opt-balanced` 直接改名或挪用为 `outputs/v0.45`。  
原因是其中的：

- `run_manifest.json`
- `release_manifest.json`
- 报告文件名
- 版本说明文本

都仍声明为 `v0.45-param-opt-balanced`，直接改目录会造成“目录名和运行元数据不一致”的错误状态。

因此，本次只完成“配置正式化”和“旧产物归档收口”，不伪造正式结果。  
`outputs/v0.45-param-opt-balanced` 在下一次正式 `v0.45` 重跑完成前，保留为当前已选正式参数的验证参考产物。

## 3. 正式入口收口方案

### 3.1 根配置

`experiment_config.yaml` 成为唯一正式人工修改入口，并满足：

- 参数容量切换到 `balanced`；
- 原 `explore` 差异值在相邻位置用注释保留；
- 注释内容能够一眼看出哪些值是正式值，哪些值是探索值。

建议注释组织方式：

- 在每个受影响段落前增加简短注释标题；
- 先放正式值；
- 紧接着放 `explore` 对应注释值；
- 不保留“旧正式参数”作为活跃注释，避免三套参数并列造成歧义。

### 3.2 `configs/experiments/`

`configs/experiments/` 不再承担 `balanced/explore` 这轮参数试验的正式入口功能。

本轮参数试验配置文件：

- `configs/experiments/v0.45_param_opt_balanced.yaml`
- `configs/experiments/v0.45_param_opt_explore.yaml`

应迁入归档区域，例如：

- `已归档/configs/experiments/`

`configs/experiments/README.md` 保留，但需要改写为：

- 说明当前正式主线已回收至根配置；
- 说明本目录只用于未来新的独立试验；
- 说明 `v0.45` 这一轮参数试验配置已归档；
- 若需要追溯，去 `已归档/configs/experiments/` 查看。

## 4. 输出与结果归档方案

### 4.1 需要归档的结果

以下目录属于旧版本或已淘汰的试验结果，应归档到 `已归档/outputs/`：

- `outputs/v0.44`
- `outputs/v0.44-param-opt-balanced`
- `outputs/v0.44-param-opt-explore`
- `outputs/v0.45-param-opt-explore`

这些目录不再属于当前正式主线，也不应继续出现在 `outputs/` 根目录下。

### 4.2 暂时保留的结果

以下目录暂时保留在 `outputs/` 根目录：

- `outputs/v0.45`
- `outputs/v0.45-param-opt-balanced`

保留原因如下：

- `outputs/v0.45` 仍是当前正式版本目录名，对项目结构与 README 示例仍有意义；
- `outputs/v0.45-param-opt-balanced` 是本轮正式参数切换的直接验证依据，在未重跑正式 `v0.45` 前仍有参考价值。

### 4.3 后续状态

后续若按新正式配置重跑 `v0.45` 并得到新的正式产物，则：

- 旧 `outputs/v0.45` 应迁入归档；
- 并将 `outputs/v0.45-param-opt-balanced` 也迁入归档，结束过渡态。

但这不属于本次收口的强制动作。

## 5. 测试与缓存归档方案

### 5.1 `tests/` 主目录保留原则

`tests/` 主目录只保留当前正式版本 `v0.45` 仍有效、仍参与默认 `pytest` 收集的测试。

这些测试必须继续满足：

- 文件名以 `test_v045_` 开头；
- 直接服务于根配置正式入口；
- 不依赖已归档的 `configs/experiments/v0.45_param_opt_*.yaml`。

### 5.2 需要处理的试验专用测试

凡是仅用于验证本轮参数试验入口存在性的测试，需要迁出主目录或删除。

重点包括：

- `tests/test_v045_experiment_config_overrides.py`

因为它显式依赖：

- `configs/experiments/v0.45_param_opt_balanced.yaml`
- `configs/experiments/v0.45_param_opt_explore.yaml`

一旦这两个文件归档，该测试就不再属于正式主线测试。

此外，需要检查并处理所有通过 `configs/experiments/v0.45_param_opt_*` 路径断言行为的测试与脚本测试：

- `tests/test_v045_run_all_script.py`
- 其他包含相同路径断言的 `test_v045_*`

处理原则：

- 如果测试的核心目的是验证正式入口行为，则改为断言根配置行为；
- 如果测试的核心目的是验证试验入口覆盖机制，则迁入 `已归档/tests/`。

### 5.3 缓存与无效文件

以下内容不应继续保留在 `tests/` 主目录：

- `tests/__pycache__/`
- 任何历史 `.pyc`
- 不再被当前测试引用的历史缓存碎片

这些内容不是“归档价值高的正式测试资产”，而是应直接清理掉的缓存噪音。

如果需要保留历史测试文件本体，则迁入：

- `已归档/tests/`

缓存文件本身不归档，直接删除。

## 7. 风险与防错要求

### 7.1 禁止误删正式结果

本次归档不能误动以下活跃目录：

- `outputs/v0.45`
- `outputs/v0.45-param-opt-balanced`

### 7.2 禁止造成配置与测试脱节

如果归档了 `configs/experiments/v0.45_param_opt_*.yaml`，就必须同步处理测试和文档，不能留下：

- README 仍指向已归档路径；
- 测试仍引用已归档文件；
- changelog 未说明入口变化。

### 7.3 禁止“目录名正确、元数据错误”

不能通过简单重命名目录让：

- `outputs/v0.45-param-opt-balanced`

伪装成：

- `outputs/v0.45`

除非内部所有元数据同步重写并经过完整验证。  
本次明确不做这一动作。

## 8. 验收标准

本次收口完成后，应满足以下条件：

1. `experiment_config.yaml` 已采用 `balanced` 正式参数，且 `explore` 只以注释保留。
2. `configs/experiments/` 不再保留本轮 `balanced/explore` 试验 YAML 活跃入口。
3. `outputs/` 根目录中不再出现 `v0.44*` 与 `v0.45-param-opt-explore`。
4. `tests/` 主目录只保留当前正式版本需要的 `test_v045_*` 测试，不再依赖已归档试验配置。
5. `tests/__pycache__/` 与历史 `.pyc` 已清理。
6. `README.md`、`configs/experiments/README.md`、`v0.45.md`、`CHANGELOG.md` 已同步说明正式入口与归档状态。
7. `pytest` 默认收集路径不再因为已归档试验入口而失败。

## 9. 本次不做的事情

以下动作不属于本次收口范围：

- 不重新训练或重跑正式 `v0.45`；
- 不把 `balanced` 结果直接改写成 `v0.45` 正式产物；
- 不新增新的参数搜索轮次；
- 不引入新的市场机制、奖励结构或实验边界。

本次工作的性质是：

- 正式入口收口
- 目录与结果治理
- 文档和测试一致性修复

而不是新的策略研究轮次。
