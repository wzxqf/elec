# v0.33 制度深结构抽取与制度建模设计草案

## 目标

v0.33 将在 v0.32 HPSO 参数化策略稳定后，单独重构政策解析与制度状态建模。目标是从当前“文件名关键词 + 少量文本关键词”的规则生成方式，升级为“文档结构解析 + 候选规则抽取 + 可审计校验 + 制度状态生成”的深结构制度建模流程。

本设计是 v0.33 的计划草案，不在 v0.32 实施。

## 当前问题

当前政策模块已经能生成文件清单、元数据索引、结构化规则表和解析失败清单，但存在以下不足：

- DOCX 解析只提取正文文本，未保留条款层级、表格、标题结构和证据片段。
- XLSX 结果表只提取少量统计字段，缺少表格级元数据。
- 规则抽取主要依赖文件名关键词，容易漏掉条款细节。
- 规则 `note` 缺少可定位的证据片段和条款编号。
- 政策状态生成缺少“候选规则 -> 审核规则 -> 生效规则”的可审计流程。

## 设计边界

### 纳入

- 深结构解析政策 DOCX、XLSX，后续可扩展 PDF。
- 保留发布时间、生效时间、条款编号、标题、正文、表格位置。
- 生成候选规则表与审核后规则表。
- 支持规则校验：时间口径、市场范围、主体范围、动作空间边界。
- 继续输出当前制度状态与四组前瞻制度状态。
- 可选接入 LLM 做候选规则抽取，但不得让 LLM 直接覆盖生效规则。

### 不纳入

- 不使用 LLM 生成政策评分。
- 不把 LLM 输出直接作为模型状态。
- 不引入论文未设定的新市场机制、新主体或新收益项。
- 不改变 v0.32 的 HPSO 参数化策略接口，只提供更高质量制度状态输入。

## 模块划分

建议新增：

```text
src/policy_deep/
  document_reader.py
  structure_extractor.py
  candidate_rules.py
  rule_review.py
  rule_validation.py
  regime_builder.py
```

### 文档读取层

输出：

- `policy_document_inventory.csv`
- `policy_document_units.csv`

核心字段：

- `document_id`
- `source_file`
- `file_name`
- `suffix`
- `publish_time`
- `unit_id`
- `unit_type`
- `heading_path`
- `paragraph_index`
- `table_index`
- `text`
- `raw_value`

### 候选规则层

输出：

- `policy_rule_candidates.csv`

核心字段：

- `candidate_id`
- `policy_name`
- `publish_time`
- `effective_start_candidate`
- `effective_end_candidate`
- `rule_type_candidate`
- `scope_candidate`
- `state_group_candidate`
- `state_name_candidate`
- `state_value_candidate`
- `evidence_unit_id`
- `evidence_text`
- `extractor`
- `confidence`
- `review_status`

候选规则可以由规则解析器、LLM 或两者组合生成。

### 审核规则层

输出：

- `policy_rule_reviewed.csv`
- `policy_rule_table.csv`

只有 `review_status = accepted` 的规则进入正式 `policy_rule_table.csv`。拒绝或需修订的候选规则保留在候选表中，不进入模型状态。

### 校验层

校验内容：

- 生效时间不得早于制度明确口径。
- 2026-01-01 前不得启用新能源机制电价执行状态。
- 2026-02-01 前不得启用年度多月交易 `40% + 60%` 价格机制。
- 规则主体必须是售电公司视角或影响售电公司收益、风险、结算的外部制度状态。
- 不得把辅助服务收益直接并入主策略收益函数。

输出：

- `policy_rule_validation_issues.csv`

## LLM 使用方式

LLM 只作为候选规则抽取器：

```text
文档结构单元 -> LLM 抽取候选规则 -> 规则校验 -> 人工或程序审核 -> 正式规则表
```

LLM 输出必须包含：

- 生效时间依据。
- 发布时间依据。
- 原文证据片段。
- 作用范围。
- 推荐 `state_name` 和 `state_value`。
- 不确定性说明。

LLM 不得：

- 输出政策打分。
- 直接写入正式规则表。
- 直接影响奖励函数。
- 引入 AGENTS.md 禁止的机制。

## 与 v0.32 的接口

v0.33 仍向主体模型提供：

- `policy_rule_table.csv`
- `policy_state_trace.csv`
- `market_rule_constraints.csv`
- `policy_parse_failures.csv`

HPSO 不直接依赖 LLM 输出，只依赖审核后的制度状态。

## 验收标准

- 每条正式规则可追溯到 `evidence_unit_id`。
- 候选规则、审核规则、正式规则分表保存。
- 时间口径校验能拦截制度前移。
- v0.32 HPSO 主流程在不启用 LLM 时仍可运行。
- 报告中说明政策来源文件数、候选规则数、接受规则数、拒绝规则数和解析失败文件数。
