# v0.32 HPSO 参数化策略重构设计

## 目标

v0.32 将当前 HPSO 从“对训练、验证、回测窗口分别离线搜索动作”的实现，重构为“训练期搜索一组可泛化策略参数，验证和回测期只加载训练后参数进行推断”的多时间尺度策略框架。

本版只处理 HPSO 主体重构、配置重构、bootstrap 生效、训练后推断、信息边界约束、滚动修正和 24 小时合约曲线的最小闭环。深结构制度抽取、LLM 条款解析和更完整市场拟真机制列入 v0.33，不混入本次主体重构。

## 当前问题

当前实现存在以下偏差：

- `experiment_config.yaml` 保留大量 PPO 字段，HPSO 只读取其中一部分参数。
- `total_timesteps`、`n_steps`、`batch_size`、`gae_lambda` 等 PPO 参数对 HPSO 主流程无实际意义。
- HPSO 在训练、验证、回测阶段都会重新对传入周集合做离线优化，验证和回测不是训练后策略推断。
- bootstrap 训练序列在 `train_hpso_model()` 中被去重，实际只使用唯一训练周。
- 上层搜索对象是每周动作，不是可泛化策略参数。
- 下层优化直接使用 `net_load_id` 构造目标，存在决策时序上的事后信息。
- 24 小时合约曲线目前由周度合约量按日前净负荷权重分配，尚未形成可解释的分时合约曲线参数。

## v0.32 范围

### 纳入

- 重构根配置为 HPSO 参数化策略专用结构。
- 新增策略参数向量 `theta`，由 HPSO 在训练期搜索。
- 将 `theta` 拆分为上层周度策略参数 `theta_U` 和下层滚动修正参数 `theta_L`。
- 训练期通过 bootstrap 序列反复 rollout，目标函数按场景聚合。
- 验证和回测期禁止重新优化目标周，只允许加载 `best_theta` 推断动作。
- 下层滚动修正只使用决策时点可见信息。
- 周度输出保留 `lock_ratio_base`、`delta_lock_ratio`、`lock_ratio_final`。
- 新增 24 小时合约曲线的最小可用参数化实现。
- 输出模型参数、参数清单、推断轨迹、信息边界审计表、训练收敛曲线。

### 不纳入

- 不引入 PPO、SAC、PSR、在线学习或独立下层 DRL。
- 不引入 LLM 政策评分系统。
- 不把需求响应、调峰辅助服务、调频市场收益纳入主策略收益函数。
- 不做 2015-2023 长历史典型场景回测。
- 不把中长期合约当作物理调度计划。

## 策略结构

### 参数维度

v0.32 默认总维度为 64：

- `theta_U`: 40 维。
- `theta_L`: 24 维。

建议结构：

| 参数块 | 维度 | 用途 |
|---|---:|---|
| `upper.lock_delta_weights` | 16 | 周度底仓残差响应 |
| `upper.bandwidth_weights` | 12 | 现货边际敞口带宽响应 |
| `upper.curve_shape_weights` | 8 | 24 小时合约曲线形状响应 |
| `upper.policy_gate_weights` | 4 | 制度状态门控 |
| `lower.spread_response` | 6 | 价差响应 |
| `lower.load_forecast_response` | 6 | 负荷预测偏差响应 |
| `lower.renewable_response` | 4 | 新能源扰动响应 |
| `lower.smoothing_response` | 4 | 小时修正平滑响应 |
| `lower.policy_band_shrink` | 4 | 制度约束收缩响应 |

该维度规模适配当前只有 13 个训练周的样本条件。直接搜索每周动作会失去泛化能力；搜索上百维策略参数会显著增加过拟合风险。

### 上层策略

每周输入状态 `s_w`，包括历史价格、负荷偏差、波动指标、制度状态、四组前瞻制度状态、新能源扰动代理、上期动作和上期奖励。策略函数输出：

```text
delta_lock_ratio_raw_w = f_U_delta(s_w; theta_U)
exposure_bandwidth_w = f_U_band(s_w; theta_U)
curve_shape_params_w = f_U_curve(s_w; theta_U)
```

最终周度底仓仍按论文口径输出：

```text
lock_ratio_final = clip_by_policy_and_week_step(
  lock_ratio_base + delta_lock_ratio,
  previous_lock_ratio
)
```

其中 `lock_ratio_base` 继续使用 `dynamic_lock_only` 强基准。

### 24 小时合约曲线

v0.32 引入最小参数化曲线，不直接建复杂月度合约撮合。

流程：

1. 由周度锁定比例确定周度中长期合约总量。
2. 生成基础 24 小时形状 `base_24h_profile`，默认来自训练集日前净负荷分时均值。
3. 使用 `curve_shape_params_w` 对峰、平、谷和新能源高扰动小时做有限形状偏移。
4. 对 24 小时曲线归一化，使一周合约量守恒。
5. 映射到每个小时，再由结算层映射到 15 分钟。

输出字段至少包括：

- `contract_hour`
- `contract_profile_base`
- `contract_profile_adjustment`
- `contract_profile_final`
- `q_lt_hourly`
- `curve_normalization_factor`

### 下层滚动修正

下层不再调用 HPSO 对整周小时修正量离线搜索。训练得到 `theta_L` 后，每小时执行：

```text
delta_q_t = f_L(
  da_price_t,
  forecast_load_t,
  forecast_renewable_t,
  historical_spread_state_t,
  policy_state_w,
  previous_delta_q,
  exposure_bandwidth_w,
  theta_L
)
```

允许使用：

- 当前日前价格。
- 当前日前净负荷预测。
- 当前日前新能源预测。
- 上一小时及更早历史统计。
- 周度制度状态和前瞻状态。
- 上一小时修正量。
- 上层给定的边际敞口带宽。

禁止用于决策：

- 当前小时 `net_load_id`。
- 当前小时日内实际价格。
- 当前小时实时偏差。
- 当周后续小时实际结果。

这些字段只用于结算、奖励和诊断。

## 训练、验证、回测

### 训练

训练入口执行：

1. 构建 bootstrap 周序列，不去重。
2. 初始化 HPSO 粒子，每个粒子代表一个完整 `theta`。
3. 对每个粒子在 bootstrap 序列上 rollout。
4. 目标函数聚合采购成本、CVaR、套保误差、交易成本、平滑性、相对 `dynamic_lock_only` 的成本差。
5. 保存最优参数到 `outputs/v0.32/models/hpso_param_policy.json`。

### 验证

验证入口只加载训练得到的 `theta`：

```text
theta -> policy inference -> weekly actions -> hourly rolling hedge -> settlement -> metrics
```

不允许调用 HPSO 优化验证周动作。

### 回测

回测与验证一致，只做策略推断和结算。基准策略仍保留：

- `fixed_lock`
- `dynamic_lock_only`
- `rule_only`
- `simple_rolling_hedge`

主报告必须重点比较 `dynamic_lock_only`。

## 配置设计

根配置继续是唯一人工入口：`experiment_config.yaml`。

v0.32 建议结构：

```yaml
project:
  version: v0.32

training:
  algorithm: HPSO_PARAM_POLICY
  seed: 42
  device: cuda
  allow_cpu: true

hpso:
  parameter_dimension: 64
  swarm:
    particles: 48
    iterations: 80
    inertia_weight: 0.70
    cognitive_factor: 1.45
    social_factor: 1.45
    initial_temperature: 1.0
    cooling_rate: 0.96
    perturbation_scale: 0.05
    stagnation_window: 10
    backprop_steps: 0
  bootstrap:
    sequence_length: 96
    block_size: 3
    scenario_repeats: 4
  objective_weights:
    procurement_cost: 1.0
    dynamic_baseline_gap: 1.15
    tail_risk: 0.55
    hedge_error: 0.18
    hourly_smooth: 0.08
    action_smooth: 0.12
    transaction: 1.0
    contract_curve_shape: 0.05

policy:
  upper:
    state_features: []
    lock_delta_scale: 0.12
    bandwidth_min: 0.0
    bandwidth_max: 0.85
  lower:
    rolling_mode: causal
    non_negative_spot: true
    smooth_limit_mwh: 420.0
  contract_curve:
    enabled: true
    periods: 24
    base_profile_source: train_da_net_load
    peak_hours: [8, 9, 10, 11, 18, 19, 20, 21]
    valley_hours: [0, 1, 2, 3, 4, 5]
```

PPO 字段应移除，或归入 `legacy_ppo` 且不被 HPSO 主流程读取。

## 验收标准

- `experiment_config.yaml` 的 `project.version` 为 `v0.32`。
- 训练阶段生成 `hpso_param_policy.json`。
- 验证和回测阶段不重新调用 HPSO 搜索目标周动作。
- bootstrap 序列长度在训练诊断中保留，且不被唯一周去重。
- `information_boundary_audit.csv` 标明每个字段是否允许进入决策。
- 下层滚动修正决策不读取 `net_load_id` 或当前日内实际价格。
- 24 小时合约曲线导出，并满足周度合约量守恒。
- 回测报告继续比较 `dynamic_lock_only` 强基准。
- 测试覆盖 HPSO 参数切分、策略推断、bootstrap 不去重、信息边界、24 小时曲线守恒和回测禁止重优化。
