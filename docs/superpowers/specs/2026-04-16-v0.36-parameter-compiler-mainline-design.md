# v0.36 Design

**Date:** 2026-04-16

**Scope:** 在不改变“双层参数化 HPSO + 小时级规则修正 + 15 分钟代理结算”正式主线的前提下，完成 `experiment_config.yaml -> 参数编译器 -> 训练核 -> 评分核` 的全链路打通，显著扩大上层/下层有效参数维度与 PSO 搜索预算，解决当前大量 agent 特征未真实进入模型的问题。

---

## 1. 背景与目标

`v0.35` 已经可以完整跑通训练、验证、回测和正式输出，但训练阶段耗时明显偏短，且当前模型结构存在以下关键断点：

1. `experiment_config.yaml` 中部分模型相关配置未真正进入 `HYBRID_PSO_V033` 主训练路径。
2. `feature_manifest` 中已选入 agent 的数值特征很多，但 `score_kernel` 仍按 `min(特征数, 粒子维度)` 截断，导致大部分特征未参与打分。
3. `upper dimension=12`、`lower dimension=8` 与当前周度/小时级特征规模严重不匹配，导致搜索空间偏小、训练预算偏保守。
4. 训练时长主要由较小的粒子规模、较少的迭代轮数和较小的有效参数空间决定，而不是由真实模型复杂度驱动。

`v0.36` 的目标是完成以下三项核心升级：

1. 新增“参数编译器”层，使 `experiment_config.yaml` 成为模型结构的唯一声明入口。
2. 取消当前隐式截断逻辑，让 agent 特征按显式参数布局进入上层/下层评分核。
3. 显著扩大有效维度和 PSO 搜索预算，拉长训练时间，并为后续版本保留可审计的模型结构扩展路径。

---

## 2. 不变边界

以下边界在 `v0.36` 继续保持不变：

- 正式主算法仍为双层参数化 HPSO 主线，不恢复 PPO / SB3 / gym 或其他上层 DRL。
- 上层仍为周度中长期持仓参数化策略，下层仍为小时级现货滚动修正规则。
- 正式结算与回测仍以 15 分钟代理结算为准。
- 制度状态仍必须区分发布时间与生效时间，不得前移应用后期政策。
- 市场范围仍限定为中长期、日前现货、日内/实时现货。
- 不引入论文主线之外的新市场主体、新市场机制、新收益项或第二套独立训练主链。
- 仍保持根目录 `experiment_config.yaml` 为唯一人工修改入口，仍保持 `run_all.py` 为人工一键入口。

`v0.36` 的工作重点是“配置驱动的参数化主链增强”，不是算法范式更换。

---

## 3. 总体方案

`v0.36` 采用 `A2` 路线：在配置与训练核之间新增一层“参数编译器”，将模型结构从“局部硬编码 + 局部配置”改为“配置声明 + 编译执行”。

新的主链如下：

`experiment_config.yaml`
-> `src/config/load_config.py`
-> `src/model_layout/compiler.py`
-> `compiled_parameter_layout`
-> `src/agents/hybrid_pso.py`
-> `src/training/score_kernel.py`

### 3.1 核心思想

- 配置文件只负责声明“哪些特征进入哪个参数块、每个参数块需要多少参数、上下层搜索预算是多少”。
- 参数编译器负责将这些声明解析成训练核可直接消费的结构化参数布局。
- 训练核只读取编译结果，不再手写或猜测真实维度。
- 评分核只按编译结果消费参数块，不再用 `min(...)` 截断。

### 3.2 预期收益

- 解决“配置写了但未生效”的结构性问题。
- 解决“特征已选入 manifest 但大部分未进入模型”的问题。
- 让维度、切片、特征映射、搜索预算全部可审计。
- 为后续版本增加维度、调整特征分组、控制训练时间提供稳定扩展点。

---

## 4. 参数编译器设计

### 4.1 新增职责

新增 `src/model_layout/compiler.py`，负责：

1. 读取运行时配置中的参数布局声明。
2. 结合 `feature_manifest`、`agent_feature_columns`、`policy_state_trace`、小时级输入字段，生成上下层参数布局。
3. 计算并返回：
   - 上层参数块列表
   - 下层参数块列表
   - 每个参数块的起止切片
   - 每个参数块对应的特征列集合
   - 最终 `upper_dim`
   - 最终 `lower_dim`
4. 对配置做强校验：
   - 引用字段不存在时报错
   - 参数块为空时报错
   - 上下层总维度为零时报错
   - 必需制度状态字段缺失时报错

### 4.2 输出结构

编译器输出统一结构化对象，例如：

- `CompiledParameterLayout`
- `UpperBlockSpec`
- `LowerBlockSpec`

至少包含：

- `upper.total_dimension`
- `lower.total_dimension`
- `upper.blocks[]`
- `lower.blocks[]`
- `feature_sources`
- `slice_map`
- `column_index_map`

训练核、评分核、报告层都只消费这一份统一结构，避免多处重复推导。

### 4.3 编译策略

参数编译器不做数值训练，只做“结构声明 -> 可执行布局”的映射。

编译后的布局应当具备以下属性：

- 顺序稳定：同一配置多次运行得到相同切片映射。
- 可导出：可写出到 `outputs/<version>/reports/compiled_parameter_layout.json`。
- 可审计：可生成 markdown 摘要，说明每个参数块消耗了哪些特征和多少维度。

## 5. `experiment_config.yaml` 结构升级

`v0.36` 中，模型结构相关配置不能再分散在“部分生效、部分失效”的多个节中。建议新增正式节：

- `parameter_compiler`

### 5.1 配置原则

- `training.algorithm` 升级为 `HYBRID_PSO_V036`
- `hybrid_pso.upper.dimension` 与 `hybrid_pso.lower.dimension` 不再手填，由编译器计算
- `upper_strategy` 与 `lower_strategy` 从“松散参考配置”升级为“参数编译输入声明”
- 必需的结构声明全部集中在 `parameter_compiler`

### 5.2 建议配置口径

#### 上层

上层建议包含以下参数块：

1. `weekly_feature_weights`
   - 来源：`agent_feature_columns`
   - 含义：对进入 agent 的周度数值特征赋予显式线性权重
2. `policy_feature_weights`
   - 来源：制度状态数值列
   - 含义：对当前制度状态与前瞻制度状态赋予显式权重
3. `contract_curve_latent`
   - 含义：定义 24 点合约曲线的形状因子，不再限制为 4 个固定因子
4. `action_head`
   - 含义：控制 `contract_adjustment`、`exposure_band`、`policy_gate` 等动作头的偏置/缩放参数

#### 下层

下层建议按小时级规则响应分块声明：

1. `spread_response`
2. `load_deviation_response`
3. `renewable_response`
4. `policy_shrink_response`

每个块都应明确：

- 输入列
- 响应参数个数
- 是否允许多列合并
- 对应的参数切片范围

### 5.3 配置审计要求

运行后必须额外输出：

- `compiled_parameter_layout.json`
- `parameter_layout_summary.md`

其中至少说明：

- 上层/下层总维度
- 每个参数块的维度
- 每个参数块消费的特征列
- 训练时实际采用的粒子数与迭代数

## 6. 有效维度扩张设计

`v0.36` 的目标不是盲目把 `dimension` 数字改大，而是让“特征数量、参数块设计、搜索预算”三者一致。

### 6.1 上层维度口径

当前 `feature_manifest` 中已被选入 agent 的数值特征约为 110 个。`v0.36` 中，上层参数至少应覆盖：

- 全部 agent 周度数值特征
- 关键制度状态数值特征
- 更丰富的 24 点曲线形状因子
- 动作头偏置/缩放参数

建议上层维度目标区间为：

- `128-160`

建议首版目标值：

- `upper_dim ≈ 140`

构成建议：

- `weekly_feature_weights`: 约 110
- `policy_feature_weights`: 约 12
- `contract_curve_latent`: 约 12
- `action_head`: 约 6

### 6.2 下层维度口径

下层仍应保持“轻量规则修正”边界，不宜无限膨胀。

建议下层维度目标区间为：

- `24-48`

建议首版目标值：

- `lower_dim ≈ 32`

构成建议：

- `spread_response`: 8
- `load_deviation_response`: 8
- `renewable_response`: 8
- `policy_shrink_response`: 8

### 6.3 必守原则

- 不允许继续出现“110 个特征进 manifest，但只消费前 12 个”的情况。
- 不允许下层为追求时长而引入脱离论文主线的复杂独立模型。
- 上层扩维必须由真实参数块驱动，而不是纯填充无语义维度。

## 7. 训练时间拉长设计

### 7.1 当前问题

当前训练时间偏短，主要由以下因素共同导致：

- `upper_particles=64`
- `lower_particles=48`
- `iterations=80`
- 有效维度过小
- 周度训练样本数较少
- 评分核为全张量化实现，吞吐很高

### 7.2 `v0.36` 搜索预算目标

在不改变 HPSO 主线的前提下，`v0.36` 通过扩大粒子规模、扩大迭代轮数和扩大有效维度来拉长训练时间。

建议首版预算：

- `upper.particles = 160`
- `lower.particles = 128`
- `upper.iterations = 240`
- `lower.iterations = 240`

### 7.3 预算控制原则

- 训练时长增加必须来自有效搜索规模增加，而不是无意义 sleep、重复 I/O 或人为拖延。
- 搜索预算提升应保留 CUDA 优先与受控 CPU 降级能力。
- 运行摘要与训练摘要中必须记录真实 `upper_dim`、`lower_dim`、粒子数与迭代数。

## 8. 训练核与评分核改造

### 8.1 `train_hybrid_pso_model`

`src/agents/hybrid_pso.py` 需要做以下调整：

- 不再从配置直接读取手写 `dimension`
- 改为读取编译后的 `upper_dim` 与 `lower_dim`
- runtime profile 中显式记录：
  - `upper_dim`
  - `lower_dim`
  - `upper_particles`
  - `lower_particles`
  - `iterations`
  - `layout_version`

### 8.2 `batch_score_particles`

`src/training/score_kernel.py` 需要做以下调整：

- 删除当前 `min(weekly_features.shape[1], upper.shape[1])` 一类隐式截断逻辑
- 改为按编译结果逐块取参数、逐块消费特征
- 上层周度信号由显式参数块构造，而不是“取前 N 个特征做一次矩阵乘”
- 下层小时级信号也按显式参数块构造
- 24 点合约曲线的形状因子数量改由编译器控制，不再固定为后 4 维

### 8.3 兼容性要求

- `materialize` 仍应能够消费训练后的上下层最优粒子
- 现有验证、回测、报告链路不应因参数布局变化被破坏
- 如需向后兼容 `v0.35` 测试夹具，应在测试层构造最小编译布局，而不是继续依赖旧截断逻辑

## 12. 验收标准

`v0.36` 完成时，至少满足以下验收条件：

1. `experiment_config.yaml` 中声明的参数布局能够真实进入训练核与评分核。
2. 不再存在“大量 agent 特征被 manifest 选中但未参与模型打分”的问题。
3. 上层真实维度提升到约 `140` 的量级，下层真实维度提升到约 `32` 的量级。
4. PSO 搜索预算显著提升，训练时间明显长于 `v0.35`。
5. `run_all.py` 仍能完整跑通训练、验证、回测与正式产物导出。
6. 参数布局可导出、可审计、可复现。

## 13. 风险与控制

主要风险包括：

1. 参数编译器与评分核之间口径不一致，导致维度错位
2. 扩维后训练成本上升过快，导致单次实验不可接受
3. 配置声明过于复杂，反而降低维护性
4. 为兼容旧测试而保留旧截断逻辑，导致主问题未真正解决

控制方式：

- 用统一 schema 结构承载编译结果，避免裸字典多处散落
- 先采用有限且有语义的参数块，不做无限自由扩维
- 明确废除旧的 `min(...)` 截断逻辑
- 将训练预算与布局摘要同时写入报告，保证变化可追踪

---

## 14. 不在 `v0.36` 范围内的内容

以下内容不纳入 `v0.36`：

- 引入新的 DRL 主链
- 引入独立下层 DRL
- 引入特征投影神经网络、MLP 编码器或更复杂函数近似器
- 改写 15 分钟代理结算主口径
- 引入论文未设定的新市场机制或新主体

`v0.36` 的定位是：先把现有参数化双层 HPSO 主线“结构打通、有效扩维、预算放大、输出可审计”。
