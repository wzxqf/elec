# v0.32 HPSO Param Policy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a v0.32 HPSO parameterized policy pipeline where training searches reusable strategy parameters and validation/backtest only perform policy inference.

**Architecture:** Add a parameterized HPSO policy layer between weekly features and backtest simulation. Training optimizes a fixed-dimensional `theta`, saves it as a model artifact, and evaluation loads that artifact without re-optimizing target weeks. Contract allocation moves from weekly load-weight allocation to a minimal 24-hour normalized contract curve.

**Tech Stack:** Python, pandas, numpy, torch, pytest, YAML configuration, existing project IO/reporting utilities.
