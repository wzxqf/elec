# v0.33 Parametric Hybrid PSO Mainline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the remaining `lock_ratio`-based mainline with the approved parametric two-layer hybrid PSO model, policy projection constraint path, `CVaR_99%`, and separated profit/reward semantics.

**Architecture:** The implementation keeps the existing formal entrypoints and the `policy_deep -> tensor_bundle -> hybrid_pso -> materialize` structure, but rewrites the strategy semantics from ratio actions to MWh-domain parameterized decisions. The training score kernel, materialization layer, and official documentation/configuration are switched together so docs, runtime config, output fields, and tests all use the same new model vocabulary.

**Tech Stack:** Python, pandas, PyTorch, pytest, Markdown, YAML
