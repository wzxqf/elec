# v0.36 Parameter Compiler Mainline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the v0.36 parameter-compiler mainline so that `experiment_config.yaml` fully defines the effective upper/lower parameter layout, the training and score kernels consume that compiled layout, and the resulting runtime is materially larger than v0.35.

**Architecture:** Introduce a dedicated model-layout layer between config loading and training. `prepare_project_context()` compiles a stable upper/lower parameter layout from config + feature sources, stores it in the runtime context, and exports audit artifacts. `train_hybrid_pso_model()` and `batch_score_particles()` stop using hard-coded dimensions and consume the compiled layout directly.

**Tech Stack:** Python, pandas, PyTorch, pytest, YAML config loading, markdown/json reporting.

from dataclasses import dataclass


@dataclass(frozen=True)
class ParameterBlockSpec:
    name: str
    columns: list[str]
    slice_start: int
    slice_end: int

    @property
    def size(self) -> int:
        return self.slice_end - self.slice_start


@dataclass(frozen=True)
class LayerLayout:
    total_dimension: int
    blocks: list[ParameterBlockSpec]


@dataclass(frozen=True)
class CompiledParameterLayout:
    upper: LayerLayout
    lower: LayerLayout
    feature_sources: dict[str, list[str]]

```python
def _block_tensor(layout: LayerLayout, block_name: str, particle_tensor: torch.Tensor) -> torch.Tensor:
    block = next(item for item in layout.blocks if item.name == block_name)
    return particle_tensor[:, block.slice_start:block.slice_end]
```
