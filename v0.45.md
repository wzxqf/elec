# elec 项目 v0.45 发布准备说明

## 一、文档目的

本文档用于说明 `v0.45` 的版本定位、本轮核心改动、验证口径与推送前状态。`v0.45` 的重点不是扩展新的论文模型边界，而是把“当前版本号、测试命名、版本派生工具和参数试验配置”统一收口到可持续迭代的口径。

---

## 二、版本定位

`v0.45` 的定位是：

> 在不改变论文主线策略边界的前提下，完成正式版本升档，并把当前版本相关文件名和规则从硬编码改为由 `project.version` 统一派生。

本版本重点解决的问题：

- 根配置虽然可以切版本，但当前主测试集文件名前缀仍需要手工维护；
- 版本报告文件名、`run_all` 解析逻辑和测试入口示例仍存在多处分散实现；
- 参数试验配置和当前正式版本之间缺少同步升档约束；
- 后续每次发版前的人工修改点仍然偏多。

---

## 三、核心改动

### 3.1 正式版本切换到 `v0.45`

- 根配置 `experiment_config.yaml` 已切换到 `project.version: v0.45`。
- README 当前版本说明、默认正式输出目录示例和测试命名说明已同步切换到 `v0.45`。
- 当前主测试集文件名前缀已从 `test_v044_` 升到 `test_v045_`。

### 3.2 版本派生工具收口

新增：

- `src/utils/versioning.py`

统一提供：

- `parse_project_version()`
- `load_project_version()`
- `build_test_prefix()`
- `build_test_filename()`
- `build_version_report_filename()`

当前以下位置已改为复用这套逻辑：

- `run_all.py`
- `tests/test_v045_repo_cleanup.py`
- `tests/test_v045_run_pytest_script.py`
- `tests/test_v045_run_all_script.py`
- `tests/test_v045_reporting_contracts.py`
- `tests/test_v045_hpso_param_config.py`
- `tests/test_v045_run_all_progress.py`

### 3.3 参数试验配置随版本升档

参数试验配置已调整为：

- `configs/experiments/v0.45_param_opt_balanced.yaml`
- `configs/experiments/v0.45_param_opt_explore.yaml`

对应 `project.version` 已同步切换到：

- `v0.45-param-opt-balanced`
- `v0.45-param-opt-explore`

### 3.4 主测试集命名规则正式生效

当前规则为：

- 仍参与 `tests/` 主目录默认收集的测试脚本，统一命名为 `test_<currentversion>_<purpose>.py`
- `<currentversion>` 由 `project.version` 去掉非字母数字字符后派生
- `tests/` 主目录不得混入 `.md`、`.txt`、`.rst` 等说明文档

相关规范已同步写入：

- `README.md`
- `AGENTS.md`
- `CHANGELOG.md`

## 四、论文主线边界

`v0.45` 不改变以下边界：

- 不改变“周度中长期底仓决策 + 小时级现货修正 + 15 分钟代理结算回测”的主结构；
- 不引入新的主训练算法；
- 不扩展新的市场主体、收益项或论文未设定动作空间；
- 不改变制度时序、生效规则与论文既定市场边界。

因此，`v0.45` 的性质仍是“工程收口与发版效率提升版”，不是研究问题重新定义版。

## 七、结论

`v0.45` 已把项目从“版本号可切换，但当前版本相关命名和路径仍需多点手工维护”的状态，推进到“版本号、测试前缀、版本报告文件名和参数试验配置可由统一版本工具协调”的状态。

后续继续升版时，仍需要执行当前测试脚本文件名的版本迁移，但代码中的版本派生逻辑、主测试约束和说明文档示例已显著收敛，发版人工修改点已减少。
