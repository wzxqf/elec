# v0.24 修改说明

## 一、本轮目标

v0.24 不改动周度动作、小时级规则、奖励结构或制度口径。本轮唯一正式目标是为当前项目补齐一个稳定、清晰、可复现的根目录一键启动入口，减少每次实验运行前的手工命令拼接和环境歧义。

---

## 二、本轮改动范围

### 1. 新增根目录一键脚本

项目根目录新增 `run_all.sh`，作为当前推荐的正式运行入口。脚本固定执行：

```bash
mamba run -n elec_env python -m src.scripts.run_pipeline
```

该设计明确约束运行环境必须为 `elec_env`，避免“当前 shell Python 环境不一致”导致的复现偏差。

### 2. 增加最小运行保护

`run_all.sh` 在正式启动前会完成以下检查：

- 是否位于项目根目录；
- `experiment_config.yaml` 是否存在；
- `mamba` 是否在当前 PATH 中可用；
- 能否从配置文件解析出 `project.version`。

脚本会在启动前打印：

- 项目根目录；
- 当前实验版本；
- 输出目录；
- 实际执行命令。

### 3. 增加 dry-run 模式

为避免每次都真正拉起训练，`run_all.sh` 增加：

```bash
bash run_all.sh --dry-run
```

该模式只打印本次版本与执行命令，不真正运行流水线，便于检查配置和入口是否正确。

### 4. 补充最小回归测试

新增 `tests/test_run_all_script.py`，验证：

- 根目录存在 `run_all.sh`；
- `--dry-run` 返回成功；
- 输出中包含固定的 `mamba run -n elec_env python -m src.scripts.run_pipeline` 命令。

---

## 三、本轮不变内容

以下内容在 v0.24 中保持不变：

- `experiment_config.yaml` 仍是唯一人工修改入口；
- 主流程入口仍然是 `src.scripts.run_pipeline`；
- 输出结构仍为 `outputs/<version>/<真实输出>`；
- 制度解析、训练、验证、回测、敏感性分析、鲁棒性分析与超参数搜索逻辑不因本轮而改变。

---

## 四、本轮交付意义

v0.24 的价值不在于新增模型能力，而在于把“如何正确启动当前整条实验链路”收敛成一个清晰、稳定、低歧义的正式入口。后续只要维护 `experiment_config.yaml` 和 `run_all.sh`，即可用一致方式重复启动同一版本实验。
