# v0.38

## 版本定位

`v0.38` 是围绕版本收口与结果解释稳定性的发布修复版。它不改变“双层参数化 HPSO + 小时级现货修正 + 15 分钟代理结算回测”的正式主线，而是修复此前阻塞主版本发布的三个问题：

- `HYBRID_PSO_V038` 无法通过主配置校验
- 训练模型元数据与运行日志仍混入旧版本号
- 政策风险调整后 Sharpe 在零波动窗口下失真

## 本版核心变化

- 正式主算法名升级为 `HYBRID_PSO_V038`
- `experiment_config.yaml` 切换到 `project.version: v0.38`
- `load_runtime_config()` 放行 `HYBRID_PSO_V038`
- `train_hybrid_pso_model()` 的模型元数据改为配置驱动版本号
- 训练/回测日志改为读取运行时版本，不再硬编码旧版
- 零波动滚动窗口的政策风险调整后 Sharpe 归零处理，避免报告中出现不可解释的极大值

## 当前正式口径

### HPSO 更新式

上下层粒子仍采用标准速度-位置更新：

- `v_{t+1} = w * v_t + c1 * r1 * (p_best - x_t) + c2 * r2 * (g_best - x_t)`
- `x_{t+1} = clip(x_t + v_{t+1}, -1, 1)`

### 上层目标

正式目标仍为最小化总负奖励：

- `score = -sum_t reward_t`
- `reward_t = excess_profit_t - lambda_tail * cvar99_t - lambda_hedge * hedge_error_t - lambda_trade * friction_cost_t - lambda_violate * policy_violation_penalty_t`

### 下层目标

小时级现货修正仍由价差、负荷偏差和新能源扰动共同驱动，并受边际敞口带宽约束：

- `spot_hedge_t = tanh(raw_hourly_signal_t) * spot_hedge_limit_t`

### 结算口径

15 分钟代理结算仍保持：

- `procurement_cost_15m = scheduled_energy_15m * (0.6 * lt_price + 0.4 * da_price) + abs(actual_energy_15m - scheduled_energy_15m) * id_price`

### 风险解释修复

`v0.38` 新增的口径仅作用于报告统计，不改动训练奖励：

- 当某滚动窗口的 `policy_risk_adjusted_excess_return_w` 标准差 `<= epsilon` 时，`window_policy_risk_adjusted_sharpe = 0`
- “持续跑赢”判定改为：窗口均值超额收益为正、胜率不少于 `0.5`，且 `(std <= epsilon 或 sharpe > 0)`

## 本版解决的问题

- 解决 `v0.38` 标签无法进入主配置加载与一键入口的问题
- 解决模型文件、日志文案和配置版本之间的漂移问题
- 解决滚动回测摘要中 Sharpe 被零波动窗口放大到不可解释量级的问题
